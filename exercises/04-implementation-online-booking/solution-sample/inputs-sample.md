# inputs サンプル

## 1. 要件メモ（抜粋）
- 利用者は予約を登録・確認・キャンセルできる
- 管理者は予約一覧を確認できる
- 今回は最小構成を優先し、通知機能は対象外にする

## 2. 仕様書（抜粋）
- 予約登録では氏名、予約日、予約時間、診療内容を入力する
- 必須項目が不足している場合はエラーとする
- 存在しない予約IDで確認した場合は見つからない旨を表示する

## 3. 設計メモ（抜粋）
- 画面：予約登録、予約確認、予約キャンセル、管理者一覧
- データ：Reservation { id, name, date, time, menu, status }
- 処理：登録、検索、取消、一覧表示

## 4. 今回の実装で特に重要なポイント
- まずは1画面で主要機能を試せるようにする
- データ保存は簡易版でよい
- エラー時は利用者向けメッセージを表示する
