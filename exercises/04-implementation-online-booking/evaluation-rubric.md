# 採点基準（5段階評価）

| 評価観点 | 説明 | 満点 |
|-----------|------|------|
| 1. 実装範囲の整理 | 何を実装し、何を対象外にしたかが明確である | 5 |
| 2. 設計とのつながり | 要件・仕様・設計を参照して実装していることが分かる | 5 |
| 3. コーディングルール設定 | ルールを設定し、実装の前提として使っている | 5 |
| 4. 動作確認 | 手動確認の結果が整理されている | 5 |
| 5. コードレビュー | ルールに基づく見直し結果が整理されている | 5 |
| 6. AI活用と見直し | AIの出力を確認し、修正・判断した形跡がある | 5 |

合計：30点満点

## 評価の目安

### 5点
- とてもよくできている
- 実装範囲が適切で、成果物のつながりが分かりやすい
- コーディングルール、動作確認、レビューまで含めて整理できている

### 3点
- 基本はできている
- 実装はあるが、レビュー結果や確認結果、説明が一部不足している
- どこをAIに任せて、どこを人が判断したかが少し見えにくい

### 1点
- まだ整理が足りない
- コードはあるが、設計とのつながりや確認結果が分からない
- AIの出力をそのまま貼っただけになっている
