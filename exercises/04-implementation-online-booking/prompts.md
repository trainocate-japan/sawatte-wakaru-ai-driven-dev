# 推奨プロンプト集（実装・Cursor活用）

この演習では、AIに一気に全部実装させるのではなく、  
**実装対象の整理 → コーディングルール設定 → 初回実装 → 修正 → レビュー**  
の順に進めます

---

## 1. 実装対象を整理する

あなたはAI駆動開発を支援するテックリードです。  
以下の要件メモ・仕様書・設計メモをもとに、初学者が実装しやすい範囲に絞ってください。

【入力資料】
（`inputs.md` の内容を貼り付け）

出力形式：
- 今回実装する機能
- 今回は実装しない機能
- 最小構成で必要な画面や処理
- 先に人が決めるべきこと
- 実装を始める順番

条件：
- 初学者向けの小さな範囲にしてください
- 予約登録、予約確認、予約キャンセル、管理者の予約一覧確認を優先してください
- 曖昧な点は「要確認」としてください

---

## 2. コーディングルール案を作る

あなたは初学者向け開発チームのテックリードです。  
以下の演習条件に合わせて、Cursor に設定するコーディングルール案を 5〜8 個作ってください。

【前提】
- 初学者向けの学習用実装
- オンライン予約システムの最小実装
- HTML / CSS / JavaScript もしくは簡単な Python Web アプリを想定

出力形式：
- ルール名
- ルール内容
- この演習でそのルールが必要な理由

条件：
- 抽象的すぎないルールにしてください
- コードレビューで確認しやすい表現にしてください
- 保守性よりも、まず「読みやすさ」「単純さ」「安全な基本動作」を重視してください

---

## 3. @メンション付きで実装を依頼する（Cursor向け）

以下は Cursor の Agent で使う想定です。

@Files inputs.md （あるいは、@Files 01~03までの　deliverables-template.md）

演習01〜03の成果物をもとに、オンライン予約システムの最小実装を作ってください。  
今回は以下を実装対象にします。
- 予約登録
- 予約確認
- 予約キャンセル
- 管理者の予約一覧確認

また、コーディングルール　（@review.mdcなど）に従ってください。

条件：
- 初学者向けに分かりやすい構成にする
- まずはローカルで動く最小構成にする
- 複雑なライブラリは増やしすぎない
- 実装前に、作成するファイル一覧と役割を先に示す
- 曖昧な点があれば実装前に確認事項として列挙する

## 5. コーディングルールに基づいてコードレビューする

あなたはAI駆動開発研修のレビュー担当です。  
コーディングルールに照らして、実装をレビューしてください。
コーディングルールは @review.mdc に従ってください。

【要件・仕様・設計】
（inputs.md の要点を貼り付け）

【実装結果】
（主要コードや成果物を貼り付け）

次の観点でレビューしてください。
1. コーディングルールに違反していないか
2. 設計と大きくずれていないか
3. 主要機能が動きそうか
4. エラー処理や入力チェックが不足していないか
5. 初学者が読める複雑さに収まっているか

出力形式：
- 良い点
- ルールに照らして修正したほうがよい点
- 優先度が高い修正
- 今回は見送ってよい修正
- 演習05で追加するとよいテスト候補
