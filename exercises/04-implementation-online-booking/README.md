# 演習04：実装（オンライン予約管理・Cursor活用）

演習01〜03で整理した **要件メモ・仕様書・設計メモ** をもとに、Cursor を使ってオンライン予約システムの最小実装を行います。  
この演習では、**ドキュメントをAIに読ませて実装する流れ** と、**Cursor のコア機能を使い分ける流れ** を体験します。

今回は第4章の内容に合わせて、主に次を扱います。
- Agent とインライン編集の使い分け
- `@` メンションによる参照資料の指定
- `.cursor/rules` などを使ったコーディングルール設定
- ルールに基づく実装とコードレビュー

テストコード作成は演習05、README などのドキュメント作成は演習06で扱います。

## ゴール
- 演習03までの成果物を入力にして、Cursor で最小実装を進められる
- Agent、インライン編集、`@` メンションを使い分けられる
- Cursor にコーディングルールを設定し、実装に反映できる
- コーディングルールに基づいて、AIにレビューを依頼し、人が採用可否を判断できる
- AIが生成したコードを読み、採用するかどうか判断できる
- 動作確認を行い、必要な修正を依頼できる

## 所要時間（約90分）
- Step1 実装対象と資料の確認（10分）
- Step2 実装方針と最小構成の決定（10分）
- Step3 Cursorにコーディングルールを設定する（15分）
- Step4 Cursorで初回実装を作る（25分）
- Step5 動作確認と修正（15分）
- Step6 コーディングルールに基づくコードレビュー（15分）

## 成果物
- `deliverables-<team>.md`：実装結果のまとめ
- 実装したアプリ一式（フォルダまたは主要ファイル）
- コーディングルールの内容（Cursor に設定した内容）
- コーディングルールに基づくレビュー結果

## この演習で作るもの
成果物には、少なくとも次の内容を含めてください。

1. 実装の対象
   - 今回実装する機能
   - 参照した要件・仕様・設計
   - 今回は実装しない範囲

2. 実装方針
   - 使用する言語や実行方法
   - 画面中心か、API中心か
   - 最小構成でどこまで作るか

3. コーディングルール
   - Cursor に設定したルール
   - そのルールを選んだ理由
   - 実装時に特に重視した点

4. 実装結果
   - 作成したファイル
   - 主要機能ごとの実装内容
   - 動作確認できたこと

5. レビュー結果
   - コーディングルールに照らしたレビュー結果
   - 修正した点
   - 今回は見送った点

6. AI活用の記録
   - どの場面でCursorを使ったか
   - うまくいった指示
   - 人が判断した点

## 手順
1. `inputs.md` に演習01〜03の成果物を貼り付ける
2. `tasks.md` に沿って演習を進める
3. 必要に応じて `prompts.md` のプロンプトを使う
4. `deliverables-<team>.md` に実装結果をまとめる
5. `evaluation-rubric.md` を使って自己評価する

## 参照
- `scenario.md`：この演習で想定する実装方針と前提
- `inputs.md`：演習01〜03の成果物を貼り付けて利用
- `tasks.md`：演習の手順と問い
- `prompts.md`：推奨プロンプト集
- `deliverables-template.md`：成果物テンプレート
- `evaluation-rubric.md`：評価観点
- `cursor-rules-template.md`：Cursor に貼り付けて使えるコーディングルール雛形
- `cursor-rules-folder-example/`：`.cursor/rules` 形式を想定したファイル構成例

## 進め方のポイント
- いきなり全部作ろうとしない
- まずは「最小構成で動くもの」を目指す
- Cursor に渡す資料は、必要なものに絞って明示する
- コーディングルールは、今回の演習で守りたい観点に絞る
- AIが書いたコードは、そのまま信用せず人が読む
- 分からないコードは採用しない
- テストやREADMEは次の演習に引き継ぐ前提で、今回は実装とレビューに集中する
