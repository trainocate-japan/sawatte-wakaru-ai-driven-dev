# シナリオ（Scenario）

あなたたちは、オンライン予約システムを段階的に開発するチームです。  
これまでの演習で、次の成果物を作成しました。

- 演習01：要件メモ
- 演習02：仕様書
- 演習03：設計メモ

この演習では、それらをもとに **Cursor を使って最小実装を作成** します。

## この演習で大切にしたいこと
- 上流工程の成果物を、実装時の入力として使うこと
- AIに丸投げせず、人が実装対象と品質を判断すること
- 演習時間内で動く最小構成に絞ること
- 実装したコードを、コーディングルールに照らして見直すこと

## 想定する進め方
- まずは 2〜4 個の主要機能に絞る
- Cursor の Agent で全体実装を依頼する
- 必要に応じてインライン編集で細かく直す
- 動作確認しながら、AIに修正を依頼する
- 最後にコードレビューを行い、テスト候補を整理する

## 今回の前提
- 完璧な業務システムを作る必要はありません
- ローカルで動く学習用の最小実装で構いません
- 難しい認証・本格DB・外部API連携は必須ではありません
- 重要なのは、**資料をもとにAIで実装を進める流れを体験すること**です
