# 演習05：テスト（オンライン予約管理・AI活用）

演習04で作成した実装をもとに、生成AIと Cursor を使って **テストコードの作成・実行・修正** を行います。  
この演習では第5章の前半に合わせて、**AIを使ってテストを効率化する流れ** を体験します。

## ゴール
- 実装コードをもとに、AIへテストコード作成を依頼できる
- 正常系と異常系の観点を整理しながらテストを増やせる
- テスト実行結果やエラーログをAIに渡して、修正サイクルを回せる
- AIが生成したテストの妥当性を人が確認できる

## 所要時間（約45分）
- Step1 テスト対象と観点を整理する（10分）
- Step2 テストコードを生成する（15分）
- Step3 テストを実行して修正する（10分）
- Step4 結果整理と振り返り（10分）

## 成果物
- `deliverables-<team>.md`：テスト結果のまとめ
- 作成したテストコード
- 実施したテスト観点一覧
- 通過・失敗・保留の記録

## 手順
1. `inputs.md` に演習04の実装内容と確認したいコードを貼り付ける
2. `tasks.md` に沿ってテストを作成・実行する
3. `prompts.md` を必要に応じて使う
4. `deliverables-<team>.md` に結果をまとめる
5. `evaluation-rubric.md` で自己評価する
