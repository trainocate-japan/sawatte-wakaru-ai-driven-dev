# タスク & タイムボックス（30分）

## Step1：ドキュメントの対象と読者を決める（5分）
- README の主な読者を決める
  - 例：研修受講者
  - 例：他チームの受講者
  - 例：初学者の開発者
- 今回 README に最低限書く項目を決める
  - アプリ概要
  - 起動方法
  - 主な機能
  - テスト方法
  - 制約や未実装事項

---

## Step2：READMEのたたき台を生成する（10分）
- 実装内容とテスト結果をAIに渡して README 作成を依頼する
- `@` メンションを使える場合は、関連ファイルを明示して渡す
- 出力形式は Markdown を指定する

---

## Step3：内容を見直して整える（10分）
- 起動方法が実際に再現できるか確認する
- 実装していない機能を書きすぎていないか確認する
- テスト方法や制約が、演習05の結果と一致しているか確認する
- 初学者が読んで分かる言葉に修正する

---

## Step4：成果物整理と共有（5分）
- `deliverables-<team>.md` に、READMEへ反映した内容を整理する
- AIが書いた下書きから、人がどこを直したか残す
- 他チームに見せる前提で、読みやすさを最後に確認する
