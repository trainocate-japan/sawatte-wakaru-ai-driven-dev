# 演習02：仕様書作成（オンライン予約管理）

演習01で整理した要件をもとに、生成AIと協働しながら**実装前に必要な仕様書**を作成します。  
この演習では、詳細な技術設計までは行わず、**画面・機能・入力/出力・業務ルールを整理して、次の設計演習につなげること**を目的とします。

「要件は分かったつもりでも、実装しようとすると足りない情報が多い」という状況はよくあります。  
この演習では、AIにたたき台を作らせながら、人が内容を確認し、**曖昧な部分を減らした仕様書**にまとめます。

## ゴール
- 要件を、実装前に必要な仕様へ具体化できる
- 画面ごとの入力、表示、操作を整理できる
- 業務ルールやエラー条件を仕様として書ける
- AIの出力を確認し、必要に応じて修正できる
- 次の設計演習で使える仕様書を完成させる

## 所要時間（約60分）
- Step1 要件の確認（10分）
- Step2 機能一覧の整理（10分）
- Step3 画面・機能仕様の作成（20分）
- Step4 業務ルール・エラー仕様の整理（15分）
- Step5 まとめ・発表（5分）

## 成果物
- `deliverables-<team>.md`：完成した仕様書

## この演習で作るもの
成果物には、少なくとも次の内容を含めてください。

1. システム概要
   - どんなシステムか
   - 誰が使うか
   - 何を実現するか

2. 機能一覧
   - 予約登録
   - 予約確認
   - 予約キャンセル
   - 管理者による予約一覧確認
   などの主要機能

3. 画面・機能仕様
   - 画面名
   - 利用者
   - 目的
   - 入力項目
   - 表示項目
   - 操作
   - 処理結果

4. 業務ルール
   - 予約できる条件
   - キャンセルできる条件
   - 入力チェック
   - 重複予約の扱い
   など

5. エラー仕様
   - 何が起きたらエラーか
   - どんなメッセージを出すか
   - 利用者にどう案内するか

## 手順
1. `tasks.md` に沿って演習を進める
2. 必要に応じて `prompts.md` のプロンプトを使う
3. 作成内容を `deliverables-<team>.md` にまとめる
4. `evaluation-rubric.md` を使って自己評価する

## 参照
- `inputs.md`：演習01の成果を貼り付けて利用
- `tasks.md`：演習の手順と問い
- `prompts.md`：推奨プロンプト集
- `deliverables-template.md`：成果物テンプレート
- `evaluation-rubric.md`：評価観点

## 進め方のポイント
- 最初から完璧な仕様書を目指さなくてよい
- 難しい技術用語より、利用者目線で分かる表現を優先する
- AIが補ってきた内容は、必ず人が妥当性を確認する
- 「この仕様書を見て、次の設計に進めるか？」を基準に考える
