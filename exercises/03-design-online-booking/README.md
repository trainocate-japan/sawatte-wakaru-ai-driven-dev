# 演習03：設計（オンライン予約管理・初学者向け）

演習02で作成した仕様書をもとに、生成AIと協働しながら**実装しやすい設計メモ**を作成します。  
この演習では、難しいアーキテクチャ設計よりも、**画面・データ・処理を整理して、次の実装演習につなげること**を重視します。

「AIにたたき台を作らせる → 人が内容を確認する → 足りない点を追加する → 実装前の設計としてまとめる」という流れを体験します。

## ゴール
- 仕様書の内容を、実装前の設計に落とし込める
- 必要な画面・データ・処理を整理できる
- AIの出力をそのまま使わず、人が確認・修正できる
- 次の実装演習で使える設計メモを完成させる

## 所要時間（約60分）
- Step1 仕様の確認（10分）
- Step2 画面設計（15分）
- Step3 データ設計（15分）
- Step4 処理設計（15分）
- Step5 まとめ・発表（5分）

## 成果物
- `deliverables-<team>.md`：完成した設計メモ

## この演習で作るもの
成果物には、少なくとも次の内容を含めてください。

1. 画面設計
   - 画面一覧
   - 各画面の入力項目・表示項目・操作
   - 画面遷移

2. データ設計
   - 必要なデータの洗い出し
   - 各データの主な項目
   - データ同士の関係

3. 処理設計
   - 予約登録
   - 予約確認
   - 予約キャンセル
   - 管理者による予約一覧確認
   などの主要処理の流れ

4. 最小API一覧（任意だが推奨）
   - どの画面から、どの処理を呼ぶかが分かる程度でよい

## 手順
1. `tasks.md` に沿って演習を進める
2. 必要に応じて `prompts.md` のプロンプトを使う
3. 設計内容を `deliverables-<team>.md` にまとめる
4. `evaluation-rubric.md` を使って自己評価する

## 参照
- `inputs.md`：演習02の成果を貼り付けて利用
- `tasks.md`：演習の手順と問い
- `prompts.md`：推奨プロンプト集
- `deliverables-template.md`：成果物テンプレート
- `evaluation-rubric.md`：評価観点

## 進め方のポイント
- 最初から完璧な設計を目指さなくてよい
- 難しい用語より、**自分たちが説明できる言葉**で書く
- AIの提案は参考にしつつ、分からない内容は採用しない
- 「この設計があると実装しやすいか？」を基準に考える
