# 演習01：要件定義（オンライン予約管理）

この演習では、生成AIと協働しながら、オンライン予約システムの**要件**を整理します。  
ここでいう要件とは、「誰のために、どんな課題を解決するシステムなのか」「何ができるとよいのか」を明確にすることです。

まだ画面やAPIを細かく決める段階ではありません。  
まずは、利用者や業務の流れを考えながら、**作るべきものの方向性を整理すること**を目的にします。

「AIに案を出してもらう → 人が妥当性を確認する → チームで優先度を決める」という流れを体験します。

## ゴール
- システムの目的と背景を説明できる
- 想定ユーザーを整理できる
- 利用者の課題と必要な機能を考えられる
- 重要な要件と優先度を整理できる
- 次の仕様書作成演習につながる要件メモを作れる

## 所要時間（約70分）
- Step1 課題の理解（10分）
- Step2 利用者と目的の整理（10分）
- Step3 必要な機能の洗い出し（15分）
- Step4 要件の優先順位づけ（15分）
- Step5：要件メモをつくる（10分）
- Step6 まとめ・発表（10分）

## 成果物
- `deliverables-<team>.md`：完成した要件メモ

## この演習で作るもの
成果物には、少なくとも次の内容を含めてください。

1. システムの目的
   - どんな課題を解決したいか
   - なぜこのシステムが必要か

2. 想定ユーザー
   - 誰が使うか
   - どんな立場の人か
   - どんな困りごとがあるか

3. 利用シーン
   - どんな場面で使うか
   - どんな流れで使うか

4. 必要な機能
   - どんなことができる必要があるか
   - 利用者向け機能、管理者向け機能

5. 要件の優先度
   - 最初に必要なもの
   - あれば便利なもの
   - 今回は対象外にするもの

## 手順
1. `tasks.md` に沿って演習を進める
2. 必要に応じて `prompts.md` のプロンプトを使う
3. 作成内容を `deliverables-<team>.md` にまとめる
4. `evaluation-rubric.md` を使って自己評価する

## 参照
- `scenario.md`：演習シナリオ
- `inputs.md`：必要に応じて追加情報を記入
- `tasks.md`：演習の手順と問い
- `prompts.md`：推奨プロンプト集
- `deliverables-template.md`：成果物テンプレート
- `evaluation-rubric.md`：評価観点

## 進め方のポイント
- 最初から完璧に決めなくてよい
- 技術の話より、まずは利用者と課題を考える
- AIの提案は参考にしつつ、違和感があれば修正する
- 「このシステムは何のためにあるのか」を説明できることを重視する
